# UX Review Notes

## Strengths

- Clear positioning: BA/PM foundation with Game Product, Game Operations, and LiveOps direction is repeated across hero, proof, and contact.
- Evidence-led structure: work samples, quantified proof stats, professional projects, and academic projects support the positioning.
- Bilingual content is centralized and structured, making future localization/design import straightforward.
- Visual system has recognizable product-work language: flow maps, device previews, permission matrix, timeline, proof stats.

## Figma Review Focus

- Preserve the dark command-center hero as the strongest first-viewport signal.
- Keep screenshots as evidence objects, not decorative backgrounds.
- Turn repeated UI into reusable components: CTA buttons, artifact cards, stat cards, logo tiles, project cards, detail blocks.
- Check mobile density: hidden secondary hero content is intentional, but the first viewport should still show name/role/CTA and a hint of product evidence.
- Review Vietnamese text expansion in card/button frames because it is longer than English in several places.

## Follow-up Improvements

- Add explicit Figma variables for color, spacing, radius, and shadow tokens.
- Create text styles for display, section title, card title, body, caption, and uppercase label.
- Build desktop first, then derive tablet/mobile frames from the same component set.
